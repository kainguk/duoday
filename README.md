# DuoDay 💗

매일의 질문과 데이트 기록을 쌓고, 원하는 기간을 한 권의 책 주문으로 묶는 커플 콘텐츠 서비스입니다.

## 서비스 소개

- **한 줄 소개**: 커플의 일상 콘텐츠(질문 답변 + 데이트 기록)를 축적하고 책으로 주문/미리보기/익스포트하는 웹앱
- **타겟 사용자**: 일상을 함께 기록하고 나중에 책으로 보관하고 싶은 커플
- **핵심 구조**: 콘텐츠가 본체(Lv1), 책 주문은 부가 기능(Lv2), 주문 데이터 익스포트(Lv3)

## 실행 방법 (Docker)

```bash
# 저장소 진입
cd duoday-main

# 환경 변수 준비
cp .env.example .env

# 실행
docker compose up --build

# 접속
http://localhost:3000
```

- DB/업로드 볼륨: `duoday-db`, `duoday-uploads`
- 최초 실행 시 마이그레이션 + 시드가 자동 반영되어 로그인 없이 확인 가능한 더미 데이터가 생성됩니다.
- 기본 더미 질문 답변 분포(최근 데이터): 둘 다 미작성 1개 / 지원만 1개 / 도윤만 1개 / 나머지 둘 다 작성
- 위 분포를 의도적으로 유지한 이유(테스트 시나리오):
  - 로그인/비로그인 모드에서 답변 노출 조건이 정확히 동작하는지 빠르게 확인하기 위함
  - **지원만 작성** 케이스: 도윤 계정으로 로그인했을 때, 본인 미작성 상태에서 지원 답변이 가려지는지 확인
  - **도윤만 작성** 케이스: 지원 계정으로 로그인했을 때, 본인 미작성 상태에서 도윤 답변이 가려지는지 확인
  - **둘 다 미작성** 케이스: 양쪽 모두 작성 UI가 정상 노출되는지 확인
  - **둘 다 작성** 케이스: 저장 후 상호 답변이 정상 노출되고 수정/삭제 후 뷰 모드 전환이 자연스러운지 확인

### 포트 변경

`docker-compose.yml`의 `ports`를 수정하세요.

```yml
ports:
  - "3100:3000"
```

그러면 `http://localhost:3100`으로 접속합니다.

## 완성한 레벨

| 레벨 | 구현 상태 | 설명 |
|---|---|---|
| Lv1 | 완료 | 오늘의 질문/지난 질문, 데이트 기록 CRUD, 다중 사진 + 대표사진, 감정 태그, 대시보드 |
| Lv2 | 완료 | 책 주문 생성/조회/상태 관리, 책 미리보기(표지/내지/마지막) |
| Lv3 | 완료 | 주문 1건 ZIP 익스포트(`order.json`, 콘텐츠 JSON, 사진 원본) |

## 주요 기능

### Lv1: 콘텐츠 서비스

- 오늘의 질문 자동 배정(라운드로빈), 답변 작성/수정/삭제
- 로그인 모드/비로그인 모드 분리, 답변 공개 조건 제어
- 데이트 기록 생성/조회/수정/삭제
- 다중 사진 업로드 + 대표사진 지정
- 감정 태그/베스트 표시, 필터링, 월별/태그 통계 대시보드
- 홈 히어로 메타에 `D+N`과 연애 시작일(예: `시작일 2024.09.14`) 표시
- 통계 카드의 "함께 답변한 질문"은 최근 8개 질문 중 두 사람이 모두 답변한 질문 수 기준

### Lv2: 책 주문 기능

- 기간/표지색/제목/포함 콘텐츠 선택 후 주문 생성
- 주문 상세/목록/상태 변경
- 상태 흐름: `pending`, `processing`, `completed`, `cancelled` (테스트용 되돌리기 포함)
- 책 미리보기(제본 여백, 내지 분할, 마지막 페이지 문구)

### Lv3: 주문 데이터 익스포트

- `/api/export/[id]`에서 주문 단위 ZIP 생성
- 포함 파일:
  - `order.json`
  - `content/questions.json`
  - `content/dates.json`
  - `content/photos/<dateId>/*`
  - `README.txt`

## 기술 스택 / 아키텍처

- **프론트/백엔드**: Next.js 14 App Router + React 18 + TypeScript
- **DB**: SQLite (`better-sqlite3`)
- **스타일**: Tailwind CSS
- **유틸**: `zod`, `nanoid`, `jszip`
- **배포/실행**: Docker, docker-compose (standalone build)

주요 디렉터리:

```text
app/
  api/
  today/
  dates/
  book/
components/
lib/
scripts/
```

## 설계 의도

- **아이디어 선택 이유**: 커플 서비스는 기록 데이터가 자연스럽게 쌓이고, 책 주문으로 확장하기 쉬운 도메인
- **사업적 가능성**: 구독형 기록 서비스 + 시즌별/기념일별 책 주문으로 LTV 확장 가능
- **추가하고 싶은 기능**:
  - 실제 계정 인증 및 권한 모델 고도화
  - 책 레이아웃 템플릿 선택/폰트 옵션
  - 사진 자동 보정/표지 추천
